# MEDMANWANN — เว็บขายไฟล์ข้อสอบจำลองออนไลน์

เว็บไซต์ขายไฟล์ข้อสอบจำลอง ลูกค้าสั่งซื้อ โอนเงิน แจ้งสลิป แอดมินตรวจสอบสลิปด้วยมือผ่านหน้าแอดมิน
แล้วปลดล็อกให้โหลดไฟล์ได้ทันที มีหน้าแอดมินจัดการสินค้า/ดูยอดขาย/ตรวจสอบและอนุมัติสลิปทุกรายการ

สร้างด้วย Next.js + Supabase (database, auth, file storage) + Resend (ส่งอีเมล)
หน้าตาออกแบบตามดีไซน์จาก Google Stitch โทนสีม่วงเข้ม/ครีม

---

## ภาพรวม: ต้องสมัครอะไรบ้าง (ทุกอันมี free tier)

| บริการ | ใช้ทำอะไร | ลิงก์สมัคร |
|---|---|---|
| Supabase | ฐานข้อมูล + ระบบล็อกอิน + เก็บไฟล์ | https://supabase.com |
| Vercel | โฮสต์เว็บไซต์ | https://vercel.com |
| Resend | ส่งอีเมลแจ้งเตือน | https://resend.com |
| GitHub | เก็บโค้ด เชื่อมกับ Vercel | https://github.com |

---

## ขั้นตอนที่ 1: สร้างโปรเจกต์ Supabase

1. ไปที่ https://supabase.com กด "Start your project" สมัครด้วยอีเมลหรือ GitHub
2. กด "New Project" ตั้งชื่อโปรเจกต์ (เช่น exam-shop) ตั้งรหัสผ่าน database (เก็บไว้ดีๆ) เลือก region สิงคโปร์ (ใกล้ไทยสุด)
3. รอสักครู่ให้ระบบสร้างโปรเจกต์เสร็จ
4. ไปที่เมนู **SQL Editor** (ไอคอนรูปดินสอด้านซ้าย) กด "New query"
5. เปิดไฟล์ `supabase/schema.sql` ในโฟลเดอร์นี้ คัดลอกทั้งหมดไปวางในช่อง SQL Editor แล้วกด **Run**
6. ถ้ารันสำเร็จ จะเห็นข้อความ "Success" — ตอนนี้มีตาราง products, orders, payment_slips ฯลฯ พร้อมระบบความปลอดภัย (RLS) ครบแล้ว
7. ไปที่เมนู **Project Settings > API** เก็บค่า 3 อย่างนี้ไว้ (จะใช้ในขั้นตอนที่ 5):
   - `Project URL` → ใช้เป็น `NEXT_PUBLIC_SUPABASE_URL`
   - `anon public` key → ใช้เป็น `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `service_role` key (กดปุ่ม "Reveal" ก่อน) → ใช้เป็น `SUPABASE_SERVICE_ROLE_KEY` (เก็บเป็นความลับ ห้ามเผยแพร่)

---

## ขั้นตอนที่ 2: สมัคร Resend (ส่งอีเมล)

1. ไปที่ https://resend.com สมัครสมาชิก
2. ไปที่เมนู **API Keys** กด "Create API Key" คัดลอกเก็บไว้
3. (ทางเลือก) ถ้ามีโดเมนตัวเองอยู่แล้ว ไปที่เมนู **Domains** เพื่อยืนยันโดเมน จะส่งอีเมลจากชื่อร้านตัวเองได้ ถ้ายังไม่มีให้ใช้ `onboarding@resend.dev` ไปก่อนได้ (ใช้ได้แค่ส่งจำกัด แนะนำให้ผูกโดเมนจริงเมื่อเปิดขายจริงจัง)

---

## ขั้นตอนที่ 3: อัปโหลดโค้ดขึ้น GitHub (ไม่ต้องใช้คำสั่ง terminal)

1. สมัครบัญชี GitHub ที่ https://github.com (ถ้ายังไม่มี)
2. กด "New repository" ตั้งชื่อ เช่น `exam-shop` เลือก Private ก็ได้ กด "Create repository"
3. ในหน้า repository ใหม่ จะเห็นปุ่ม **"uploading an existing file"** กดปุ่มนั้น
4. ลากไฟล์/โฟลเดอร์ทั้งหมดในโปรเจกต์นี้ (ยกเว้น `node_modules` ถ้ามี — ในที่นี้ยังไม่มีเพราะยังไม่ได้ install) ไปวางในหน้าอัปโหลด
5. กด "Commit changes" ด้านล่าง

> หมายเหตุ: ไฟล์ `.env.example` จะถูกอัปโหลดไปด้วย (ไม่มีรหัสลับจริงอยู่ในนั้น ปลอดภัย) แต่ห้ามสร้างไฟล์ `.env.local` ที่มีรหัสจริงแล้วอัปโหลดขึ้น GitHub เด็ดขาด — ไฟล์ `.gitignore` ป้องกันไว้แล้วถ้าใช้ git แบบคำสั่ง แต่ถ้าอัปโหลดมือผ่านเว็บต้องเช็คเองว่าไม่ได้ลากไฟล์ .env.local ขึ้นไปด้วย

---

## ขั้นตอนที่ 4: Deploy ขึ้น Vercel

1. ไปที่ https://vercel.com สมัครสมาชิกโดยกด **"Continue with GitHub"** (ให้สิทธิ์ Vercel เข้าถึง repo)
2. กด "Add New..." > "Project"
3. เลือก repository `exam-shop` ที่อัปโหลดไว้ กด "Import"
4. ในหน้า "Configure Project" จะมีช่อง **Environment Variables** — ใส่ค่าทั้งหมดนี้ (ชื่อซ้ายตรงกับใน `.env.example`):

   | ชื่อตัวแปร | ค่าที่ใส่ |
   |---|---|
   | `NEXT_PUBLIC_SUPABASE_URL` | จาก Supabase ขั้นตอนที่ 1 |
   | `NEXT_PUBLIC_SUPABASE_ANON_KEY` | จาก Supabase ขั้นตอนที่ 1 |
   | `SUPABASE_SERVICE_ROLE_KEY` | จาก Supabase ขั้นตอนที่ 1 |
   | `RESEND_API_KEY` | จาก Resend ขั้นตอนที่ 2 |
   | `RESEND_FROM_EMAIL` | `onboarding@resend.dev` (หรือโดเมนตัวเอง) |
   | `ADMIN_NOTIFICATION_EMAIL` | อีเมลของคุณที่จะรับการแจ้งเตือน |

5. กด **Deploy** รอสักครู่ (1-3 นาที) ระบบจะ build และขึ้นเว็บให้อัตโนมัติ
6. เมื่อเสร็จจะได้ลิงก์เว็บ เช่น `exam-shop-xxxx.vercel.app` — เปิดดูได้ทันที

---

## ขั้นตอนที่ 5: สร้างบัญชีแอดมินคนแรก

1. เข้าเว็บที่ deploy แล้ว กด "สมัครสมาชิก" สมัครด้วยอีเมลของคุณ (เจ้าของร้าน) ตามปกติ
2. ยืนยันอีเมล (เช็คอินบ็อกซ์)
3. กลับไปที่ Supabase Dashboard > SQL Editor > New query รันคำสั่งนี้ (เปลี่ยนอีเมลเป็นของคุณ):

```sql
update public.profiles set is_admin = true
where id = (select id from auth.users where email = 'your-admin-email@example.com');
```

4. ล็อกอินเข้าเว็บใหม่อีกครั้ง จะเห็นเมนู "แดชบอร์ดแอดมิน" ปรากฏที่แถบบนของเว็บ

---

## ขั้นตอนที่ 6: เพิ่มสินค้าแรก

1. ไปที่ `/admin/products` กด "+ เพิ่มสินค้าใหม่"
2. กรอกชื่อ ราคา อัปโหลดไฟล์ข้อสอบ (PDF) และรูปหน้าปก กด "เพิ่มสินค้า"
3. กลับไปหน้าแรกของเว็บจะเห็นสินค้าโชว์ขึ้นมาทันที

---

## ขั้นตอนที่ 7: ตั้งเลขบัญชีธนาคารจริง

ตอนนี้เลขบัญชีในหน้าชำระเงินยังเป็นค่าตัวอย่าง (XXX-X-XXXXX-X) ต้องแก้เป็นเลขบัญชีจริงของคุณ:

1. เปิดไฟล์ `src/app/(shop)/orders/[id]/page.tsx`
2. หาส่วนนี้แล้วแก้เป็นข้อมูลจริง:
   ```
   <p>ธนาคาร: กสิกรไทย</p>
   <p>เลขที่บัญชี: XXX-X-XXXXX-X</p>
   <p>ชื่อบัญชี: บริษัท ตัวอย่าง จำกัด</p>
   ```
3. ถ้าไม่สะดวกแก้โค้ดเอง บอกผมได้เลย ผมแก้ให้แล้วส่งไฟล์ใหม่ให้อัปโหลดทับ

ถ้าอยากใส่ QR พร้อมเพย์ด้วย แนะนำสร้างรูป QR (จากแอปธนาคาร) แล้วอัปโหลดเป็นรูปภาพแทนข้อความ — บอกผมได้เช่นกัน

## วิธีดูแลระบบหลังเปิดใช้งานจริง

- **ดูยอดขาย**: เข้า `/admin` จะเห็นยอดขายรวมและจำนวนคำสั่งซื้อ
- **ตรวจสอบสลิปที่ลูกค้าแจ้งเข้ามา**: ทุกสลิปจะไปอยู่ที่ `/admin/orders` ในส่วน "รอตรวจสอบสลิปด้วยมือ" คุณดูรูปสลิป เทียบกับยอดเงินที่ต้องชำระ แล้วกด "อนุมัติ" (ปลดล็อกไฟล์ให้ลูกค้าทันที) หรือ "ปฏิเสธ" ได้เลย
- **เพิ่ม/ปิดการขายสินค้า**: ที่ `/admin/products`
- **อีเมลแจ้งเตือน**: ทุกครั้งที่ลูกค้าแจ้งสลิปใหม่ จะมีอีเมลส่งเข้า `ADMIN_NOTIFICATION_EMAIL` ให้เข้าไปตรวจสอบ และทุกครั้งที่อนุมัติสำเร็จ ลูกค้าจะได้รับอีเมลแจ้งพร้อมลิงก์ดาวน์โหลด

---

## ถ้าอยากได้โดเมนของตัวเอง (.com)

1. ซื้อโดเมนจาก Namecheap, GoDaddy หรือซื้อผ่าน Vercel ได้เลย (ประมาณ 300-500 บาท/ปี)
2. ใน Vercel ไปที่ Project > Settings > Domains ใส่โดเมนที่ซื้อมา
3. Vercel จะบอกให้ไปตั้งค่า DNS ที่ผู้ขายโดเมน (ทำตามคำแนะนำในหน้านั้นได้เลย ใช้เวลาไม่กี่นาทีถึงไม่กี่ชั่วโมงกว่าจะเริ่มใช้งานได้)

## โครงสร้างโปรเจกต์ (ไว้อ้างอิงเผื่อถามต่อ)

```
src/
  app/
    (shop)/                -- หน้าฝั่งลูกค้าทั้งหมด (มี Navbar/Footer ของร้าน)
      page.tsx              -- หน้าแรก (landing page)
      products/             -- หน้ารายการสินค้า
      products/[id]/        -- หน้ารายละเอียดสินค้า
      orders/[id]/          -- หน้าออเดอร์ (แจ้งสลิป/ดาวน์โหลด)
      login/, signup/       -- หน้าเข้าสู่ระบบ/สมัครสมาชิก
    admin/                  -- หน้าแอดมินทั้งหมด (มี sidebar ของตัวเอง แยกจากฝั่งลูกค้า)
    api/                    -- API endpoints ทั้งหมด (สร้างออเดอร์, อัปโหลดสลิป, ดาวน์โหลดไฟล์)
  components/                -- React components ที่ใช้ซ้ำ
  lib/                       -- โค้ดเชื่อม Supabase, Resend, utility functions
supabase/schema.sql          -- SQL สร้างฐานข้อมูลทั้งหมด
```

หมายเหตุ: โฟลเดอร์ `(shop)` มีวงเล็บล้อมอยู่ เรียกว่า "route group" ของ Next.js
ไม่กระทบ URL จริงเลย (เช่น `/products` ก็ยังเป็น `/products` เหมือนเดิม)
ใช้แค่จัดกลุ่มหน้าที่ต้องมี Navbar/Footer เดียวกัน แยกจากโซนแอดมินที่มี sidebar ของตัวเอง

## ติดขัดตรงไหน ถามได้เลย
